Change Calculation


Max is a cashier at the local bakery. When customers pay with various coins and bills, he needs to quickly calculate the total amount he needs to return to them based on the denominations provided.


The available denominations are 25 cent, 50 cent, 1 dollar, 5 dollar, 10 dollar, and 20 dollar.


Your task is to calculate the total amount Max will return based on the number of coins and bills provided by the customer.


NOTE: 100 cent equals 1 dollar.




Input Format


    

  • The input consists of a single line with 6 space-separated integers representing the count of each denomination from 25 cent to 20 dollar.
    • 



Output Format


    

  • Output the total money that needs to be returned in dollars and cents.
    • 





Constraints


    

  • Total amount of Money <= 100000
    • 

  • Count of each denomination <= 100
    • 





Example 1


Input


6 3 2 0 5 7




Output


195 dollar 0 cent




Explanation


-> Count of 25 cent is 6 => 256 = 150 cent = 1 dollar and 50 cent
-> Count of 50 cent is 3 => 503 = 150 cent = 1 dollar and 50 cent
-> Count of 1 dollar is 2 => 12 = 2 dollar
-> Count of 5 dollar is 0 => 50 = 0 dollar
-> Count of 10 dollar is 5 => 105 = 50 dollar
-> Count of 20 dollar is 7 => 207 = 140 dollar


Total money = (50 cent + 50 cent) + 1 + 1 + 2 + 0 + 50 + 140 = 195 dollar and 0 cent


counts_str=input().split()
counts=[int(c) for c in counts_str]

denominations_cents=[25,50,100,500,1000,2000]
total_cents=0 
for i in range(len(counts)):
    total_cents+=counts[i]*denominations_cents[i]
dollars=total_cents//100
cents=total_cents%100
print(f"{dollars} dollar {cents} cent")

Write a program to generate binary representations for integers within a specified range a and b, ensuring that each binary number has a length of at least 3 digits and does not contain consecutive 0's.
The range specified by a and b is exclusive.




Input Format


    

  • First line contains an integer a.
    • 

  • Second line contains an integer b.
    • 





Output Format


    

  • The output consists of binary numbers separated by space, which follows the specified conditions.
    • 





Constraints


    

  • 1 <=a





Example 1


Input


1 
10




Output


101 110 111




Explanation


The binary numbers from the range 1-10 whose length is greater than or equal to 3 and do not contain consecutive 0's are:
1 ->   1  (length is not >= 3)
2 -> 10 (length is not >= 3)
3 -> 11  (length is not >= 3)
4 -> 100 (contains consecutive 0's)
5 -> 101
6 -> 110
7 -> 111
8 -> 1000 (contains consecutive 0's)
9 -> 1001 (contains consecutive 0's)


a=int(input())
b=int(input())
valid_binary_numbers=[]
for num in range(a+1,b):
    binary_str=bin(num)[2:]
    if len(binary_str)>=3 and "00" not in binary_str:
        valid_binary_numbers.append(binary_str)
print(" ".join(valid_binary_numbers))

Multi-Purpose Bank Account System


Create a banking system with an Account superclass and two subclasses: InvestmentAccount and LoanAccount.


Class Design


Account


    

  • Attributes
    • 

  • account_holder – Name of the account owner.
    • 

  • balance – Current balance in the account.
    • 

  • transactions – List storing all transaction details.
    • 

  • Methods
    • 

  • deposit(amount) – Increases balance and logs "Deposited: [amount]".
    • 

  • withdraw(amount) – Deducts amount if possible, logs "Withdrew: [amount]", otherwise prints "Insufficient funds".
    • 

  • check_balance() – Returns the current balance.
    • 

  • view_transactions() – Displays the transaction history.
    • 



InvestmentAccount (inherits from Account)


    

  • Additional Attributes
    • 

  • growth_rate – Growth rate percentage per year (e.g., 4.5 for 4.5%).
    • 

  • Additional Methods
    • 

  • calculate_growth() – Uses:
    
    growth = balance * growth_rate / 100
    
    Adds the growth amount and logs "Investment Growth: [growth_amount]".
    • 



LoanAccount (inherits from Account)


    

  • Additional Attributes
    • 

  • loan_limit – Maximum allowed negative balance (e.g., 5000).
    • 

  • Overridden Methods
    • 

  • withdraw(amount) – Allows overdraft within loan_limit, else prints "Loan limit exceeded".
    • 

  • If balance turns negative, logs "Withdrew: [amount] (Loan Taken)".
    • 




1. Account constructor initializes account_holder, balance, and an empty transactions list.
2. InvestmentAccount requires growth_rate in initialization.
3. LoanAccount requires loan_limit in initialization.
4. Every transaction must be recorded.
5. Proper method overriding and implementation will be assessed.

class Account:
    def __init__(self,account_holder,balance):
        self.account_holder=account_holder
        self.balance=balance
        self.transactions=[]
    def deposit(self,amount):
        self.balance+=amount
        self.transactions.append(f"Deposited: {amount}")
    def withdraw(self,amount):
        if self.balance>=amount:
            self.balance-=amount
            self.transactions.append(f"Withdrew: {amount}")
        else:
            print("Insufficient funds")
    def check_balance(self):
        return self.balance
    def view_transactions(self):
        for transaction in self.transactions:
            print(transaction)
class InvestmentAccount(Account):
    def __init__(self,account_holder,balance,growth_rate):
        super().__init__(account_holder,balance)
        self.growth_rate=growth_rate
    def calculate_growth(self):
        growth_amount=self.balance*self.growth_rate/100
        self.balance+=growth_amount
        self.transactions.append(f"Investment Growth: {growth_amount}")
class LoanAccount(Account):
    def __init__(self,account_holder,balance,loan_limit):
        super().__init__(account_holder,balance)
        self.loan_limit=loan_limit
    def withdraw(self,amount):
        if self.balance-amount>=-self.loan_limit:
            self.balance-=amount
            self.transactions.append(f"Withdrew: {amount}")
        if self.balance<0:
            self.transactions.append("(LoanTaken)")
        else:
            print("Loan limit exceeded")

Comfortable Number


Given an array of integers A, and an integer D find number of Comfortable numbers.


Element X is Comfortable if there exists at least 1 element whose absolute difference with X is less than or equal to D i.e. an element X is Comfortable, if there is another element in the range [X-D, X+D] other than X itself.




Input Format


    

  • First line contains two integers N and D respectively.
    • 

  • Second line contains N integers separated by space.
    • 





Output Format


    

  • Print a single integer denoting the total number of comfortable elements.
    • 





Constraints


    

  • 1 <=N<= 1000
    • 

  • 0 <=D<= 1000
    • 

  • 0 <=A[i]<= 1000
    • 





Explanation


Let's take an example:


4 5
10 21 13 14


arr = [10, 21, 13, 14]


For X=10 to be Comfortable element, 
there should be atleast one element in range: [10-5, 10+5] = [5, 15].
So, this is a Comfortable element. 


For X=21 to be Comfortable element, 
there should be atleast one element in range: [21-5, 21+5] = [16, 26].
There is no element in this range.
So, this is a not Comfortable element.


Similarily,
13 and 14 are Comfortable element 
Hence we have 3 Comfortable element.


import bisect
N,D=map(int,input().split())
A=list(map(int,input().split()))
A.sort()
comfortable_count=0
for x in A:
    lower_bound=x-D
    upper_bound=x+D 
    idx_low=bisect.bisect_left(A,lower_bound)
    idx_high=bisect.bisect_right(A,upper_bound)
    
    if (idx_high-idx_low)>1:
        comfortable_count+=1
print(comfortable_count)
