Remove Three Consecutive Zeros with Minimum Length 5


You are given two integers a and b.
Write a program to generate binary representations for integers within a specified range a and b, ensuring that each binary number has a length of at least 5 digits and does not contain three consecutive 0's.
The range specified by a and b is exclusive.




Input Format


    

  • First line contains an integer a.
    • 

  • Second line contains an integer b.
    • 





Output Format


    

  • The output consists of binary numbers separated by space, which follows the specified conditions.
    • 





Constraints


    

  • 1 <=a





Example 1


Input


17
20




Output


10010 10011




Explanation


The binary numbers from the range 17-20 whose length is greater than or equal to 5 and do not contain three consecutive 0's are:
18 -> 10010
19 -> 10011


#Write your code here
a = int(input())
b = int(input())
found = False
for num in range(a+1, b):
    binary = bin(num)[2:]
    if len(binary) >= 5 and '000' not in binary:
        print(binary, end=' ')
        found = True 
if not found:
    print()

- If two characters have the same frequency, select the one with the lower ASCII value.
- The character replacing_char is between 'a' and 'z'.
- All characters in sequence are lowercase.




Input Format


The input comprises two lines:


    

  • The first line contains the string sequence.
    • 

  • The second line contains the character replacing_char.
    • 





Output Format


Output the modified string after making the replacements.




Constraints


    

  • 0 <= length of string <= 10^5
    • 





Example


Input:


jkjkjkabc
h


Output:


hkhkhkabc


Explanation:
In the string sequence, 'k' and 'j' occurs most frequently but 'j' has lower ascii value. Replacing it with 'h' gives hkhkhkabc.

Proximate Number


Given an array of integers F, and an integer J find the number of Proximate Numbers.


Element V is Proximate if there is at least 1 element whose absolute difference with V is less than or equal to J. Any element V is considered Proximate, if there is another element in the range [V-J, V+J] other than V itself.




Input Format


    

  • First line contains two integers R and J respectively.
    • 

  • Second line contains R integers separated by space.
    • 





Output Format


    

  • Print a single integer denoting the total number of proximate elements.
    • 





Constraints


    

  • 1 <=R<= 1000
    • 

  • 0 <=J<= 1000
    • 

  • 0 <=F[i]<= 1000
    • 





Explanation


Let's take an example:


4 5
10 21 13 14


arr = [10, 21, 13, 14]


For V=10 to be proximate element, 
there should be atleast one element in range: [10-5, 10+5] = [5, 15].
So, this is a proximate element. 


For V=21 to be proximate element, 
there should be atleast one element in range: [21-5, 21+5] = [16, 26].
There is no element in this range.
So, this is a not proximate element.


Similarily,
13 and 14 are proximate element 
Hence we have 3 proximate element.

n, k = map(int, input().split())

E-Wallet Transaction System


Design a digital banking system with a base class EWallet and two subclasses RewardWallet and OverdraftWallet.


Class Breakdown


EWallet


    

  • Attributes
    • 

  • user – Account owner's name.
    • 

  • funds – Available balance (float).
    • 

  • logs – List to store transaction records.
    • 

  • Methods
    • 

  • load_money(amount) – Adds amount to funds and logs "Loaded: [amount]".
    • 

  • make_payment(amount) – Deducts amount if possible, logs "Paid: [amount]", else prints "Insufficient funds".
    • 

  • get_funds() – Returns available balance.
    • 

  • transaction_history() – Displays all transactions.
    • 



RewardWallet (inherits from EWallet)


    

  • Additional Attributes
    • 

  • bonus_rate – Reward percentage (e.g., 3 for 3%).
    • 

  • Additional Methods
    • 

  • apply_bonus() – Uses:
    
    bonus = funds * bonus_rate / 100
    
    Adds the bonus and logs "Bonus Applied: [bonus_amount]".
    • 



OverdraftWallet (inherits from EWallet)


    

  • Additional Attributes
    • 

  • limit – Allowed negative balance (e.g., 500).
    • 

  • Overridden Methods
    • 

  • make_payment(amount) – Allows overdraft within the limit, else prints "Overdraft limit exceeded".
    • 

  • If balance is negative, logs "Paid: [amount] (Overdraft Used)".
    • 




1. EWallet constructor initializes user, funds, and an empty logs list.
2. RewardWallet requires bonus_rate in initialization.
3. OverdraftWallet requires limit in initialization.
4. Transaction records must be maintained.
5. Correctness of features and method overriding will be tested.

class EWallet:
    def __init__(self, user, funds):
        self.user = user
        self.funds = float(funds)
        self.logs = []
    def load_money(self, amount):
        self.funds+=amount
        self.logs.append(f"Loaded: {amount}")
    def make_payment(self, amount):
        if self.funds >= amount:
            self.funds -= amount 
            self.logs.append(f"Paid: {amount}")
        else:
            print("Insufficient funds")
    def get_funds(self):
        return self.funds
    def transaction_history(self):
        for entry in self.logs:
            print(entry)
class RewardWallet(EWallet):
    def __init__(self, user, funds, bonus_rate):
        super().__init__(user, funds)
        self.bonus_rate = bonus_rate
    def apply_bonus(self):
        bonus = self.funds * self.bonus_rate / 100 
        self.funds += bonus 
        self.logs.append(f"Bonus Applied: {bonus}")
class OverdraftWallet(EWallet):
    def __init__(self, user, funds, limit):
        super().__init__(user, funds)
        self.limit = limit
    def make_payment(self, amount):
        if self.funds - amount >= -self.limit:
            self.funds -= amount
            if self.funds < 0:
                self.logs.append(f"Paid: {amount} (Overdraft Used)")
            else:
                self.logs.append(f"Paid: {amount}")
        else:
            print("Overdraft limit exceeded")
