Transport


    

  • Private Attributes:

    • 

  • __type – The type of transport (e.g., car, bike, bus).

    • 

  • 

    __distance_covered – The total distance covered in kilometers.
    

    • 

  • 

    Public Methods:
    

    • 

  • get_brand() – Returns the transport's brand.
    • 

  • set_brand(brand) – Updates the transport's brand.
    • 

  • get_type() – Returns the transport type.
    • 

  • set_type(type) – Updates the transport type.
    • 

  • get_year() – Returns the transport's manufacturing year.
    • 

  • set_year(year) – Updates the transport's manufacturing year. If the year is before 1800 (before modern transport), it should print "Invalid Year".
    • 

  • get_distance_covered() – Returns the distance covered.
    • 

  • set_distance_covered(distance) – Updates the distance covered. If the distance is negative, it should print "Invalid Distance".
    • 

  • display_info() – Prints the transport's details in the following format:
    
    "Transport [brand] [type], Year: [year], Distance Covered: [distance_covered] km"
    • 



ElectricTransport (Inherits from Transport)


    

  • Additional Private Attributes:
    • 

  • __battery_size – The battery size of the electric transport in kWh.
    • 

  • 

    __charge_duration – The time required to fully charge the battery in hours.
    

    • 

  • 

    Public Methods:
    

    • 

  • get_battery_size() – Returns the battery size.
    • 

  • set_battery_size(size) – Updates the battery size. If the size is negative, it should print "Invalid Battery Size".
    • 

  • get_charge_duration() – Returns the charge duration.
    • 

  • set_charge_duration(duration) – Updates the charge duration. If the duration is negative, it should print "Invalid Charge Duration".
    • 

  • display_info() – Overrides the display_info() method to include battery size and charge duration in the following format:
    "Electric Transport [brand] [type], Year: [year], Distance Covered: [distance_covered] km, Battery Size: [battery_size] kWh, Charge Duration: [charge_duration] hours"
    • 




1. The Transport class should have an init method that initializes all private attributes using parameters: brand, type, year, and distance_covered.
2. The ElectricTransport class should inherit from the Transport class and add new private attributes for battery_size and charge_duration.
3. The ElectricTransport class should override the display_info() method to include the battery size and charge duration.
4. Ensure that year updates for both Transport and ElectricTransport enforce the condition of being 1800 or later.
5. Ensure that distance updates for both Transport and ElectricTransport enforce the condition of being non-negative.
6. Ensure that battery size and charge duration are valid (non-negative values) for ElectricTransport.
7. Testing will verify that both Transport and ElectricTransport classes function correctly with inheritance and that the new electric transport functionality works.

  • Private Attributes:
    • 

  • __name – The name of the player.
    • 

  • __age – The age of the player.
    • 

  • __team – The team the player belongs to.
    • 

  • 

    __base_score – The base score of the player, which is a default score that applies to all players.
    

    • 

  • 

    Public Methods:
    

    • 

  • display_info(self) – This method should print the player's name, age, team, and base score in the format:
    
    Name: [name], Age: [age], Team: [team], Base Score: [base_score]
    • 

  • calculate_performance_score(self) – This method should calculate and return the performance score based on the player's type. The base score is used as the starting point.
    • 





Striker (inherits from Player)


    

  • Additional Attributes:
    • 

  • 

    __goals_scored – The number of goals the striker has scored.
    

    • 

  • 

    Overrides calculate_performance_score method:
    

    • 

  • The performance score for a striker is calculated as:
    Performance Score = Base Score + (Goals Scored * 3)
    • 





Goalkeeper (inherits from Player)


    

  • Additional Attributes:
    • 

  • 

    __saves_made – The number of saves the goalkeeper has made.
    

    • 

  • 

    Overrides calculate_performance_score method:
    

    • 

  • The performance score for a goalkeeper is calculated as:
    Performance Score = Base Score + (Saves Made * 1.5)
    •

2. The calculate_performance_score(self) method in Striker should calculate the performance score using the formula:

   Performance Score = Base Score + (Goals Scored * 3)
3. The calculate_performance_score(self) method in Goalkeeper should calculate the performance score using the formula:

   Performance Score = Base Score + (Saves Made * 1.5)
4. The calculate_performance_score(self) method in Player should return the base score if not overridden by subclasses.
5. Testing will verify that polymorphism is correctly implemented and that method overriding works as specified.

    • 

  • Methods
    • 

  • add_funds(amount) – Increases balance by amount and logs "Added: [amount]".
    • 

  • pay(amount) – Deducts amount if possible, logs "Spent: [amount]", else prints "Insufficient balance".
    • 

  • check_balance() – Returns current balance.
    • 

  • transaction_log() – Displays all recorded transactions.
    • 



LoyaltyWallet (inherits from DigitalWallet)


    

  • Additional Attributes
    • 

  • reward_percentage – Cashback percentage (e.g., 5 for 5%).
    • 

  • Additional Methods
    • 

  • apply_cashback() – Uses:
    
    cashback = balance * reward_percentage / 100
    
    Adds cashback and logs "Cashback Applied: [cashback_amount]".
    • 



CreditWallet (inherits from DigitalWallet)


    

  • Additional Attributes
    • 

  • credit_limit – Maximum negative balance allowed (e.g., 1000).
    • 

  • Overridden Methods
    • 

  • pay(amount) – Allows transactions within credit_limit, else prints "Credit limit exceeded".
    • 

  • If balance goes negative, logs "Spent: [amount] (Credit Used)".
    •

2. LoyaltyWallet requires reward_percentage during initialization.
3. CreditWallet requires credit_limit during initialization.
