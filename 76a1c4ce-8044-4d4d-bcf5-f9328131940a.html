    •

    • 

  • transactions – Stores the transaction history.
    • 

  • Methods
    • 

  • deposit(amount) – Increases balance and logs "Deposited: [amount]".
    • 

  • pay(amount) – Deducts amount if possible, logs "Paid: [amount]", otherwise prints "Insufficient balance".
    • 

  • get_balance() – Returns current balance.
    • 

  • transaction_log() – Prints transaction history.
    • 



DigitalCryptoWallet (inherits from EWallet)


    

  • Additional Attributes
    • 

  • crypto_bonus_rate – Bonus percentage for maintaining balance (e.g., 5 for 5%).
    • 

  • Additional Methods
    • 

  • apply_bonus() – Uses:
    
    bonus = balance * crypto_bonus_rate / 100
    
    Adds the bonus and logs "Crypto Bonus Added: [bonus_amount]".
    • 



PrepaidEWallet (inherits from EWallet)


    

  • Additional Attributes
    • 

  • overdraft_limit – Maximum negative balance allowed (e.g., 500).
    • 

  • Overridden Methods
    • 

  • pay(amount) – Allows payments within the overdraft limit, else prints "Overdraft limit exceeded".
    • 

  • If balance turns negative, logs "Paid: [amount] (Overdraft Used)".
    • 




1. EWallet initializes user_name, balance, and sets transactions as an empty list.
2. DigitalCryptoWallet requires crypto_bonus_rate in initialization.
3. PrepaidEWallet requires overdraft_limit in initialization.
4. All transactions must be recorded in the log.
5. Proper implementation of method overriding and correctness will be tested.
