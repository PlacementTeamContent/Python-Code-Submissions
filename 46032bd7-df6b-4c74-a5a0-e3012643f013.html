Username Encoding


You are given a username string U of length N. The username needs to be encoded before storing it in the system. The username string can be encoded using the following rules:


    

  • If the k-th character in U is a vowel, change it to the third next character in the alphabetical sequence. For example, the third next character of o is r.
    • 

  • If the k-th character in U is a consonant, change it to the third previous character in the alphabetical sequence. For example, the third previous character of h is e.
    • 



Implement the function encode which will take username string U as argument and return encoded string.



1) The third previous character of c is z, b is y.



Note: You need not to write code to handle input and output.

    • 





Output Format

def encode(U):
    a="abcdefghijklmnopqrstuvwxyz"
    x=""
    for i in U:
        if i in ["a","e","i","o","u"]:
            b=a.index(i)+3
            x=x+a[b]
        else:
            c=a.index(i)-3
            x=x+a[c]
    return(x)

Good Number Sum


For a list of integers nums, nums[i] is said to be Good Number if parity of i and nums[i] will be same.


Implement the function sum_good_number which will take n, size of list and nums as argument and return an integer, sum of good number in the list.


Note: You need not to write code to handle input and output.




Input Format


    

  • First line contains an integer n, the size of nums.
    • 

  • Second line contains n space-separated integers, numbers in list.
    • 





Output Format


    

  • One line containing an integer, the sum of good numbers.
    • 





Example 1


Input


3
1 2 3




Output


0




Explanation


    

  • For i = 0, nums[0] = 1, 1 % 2 = 1 and i % 2 = 0. Not a good number.
    • 

  • For i = 1, nums[1] = 2, 2 % 2 = 0 and i % 2 = 1. Not a good number.
    • 

  • For i = 2, nums[2] = 3, 3 % 2 = 1 and i % 2 = 0. Not a good number.
    • 



Hence, the answer is 0.


Example 2


Input


4
2 3 4 5




Output


14




Explanation


    

  • For i = 0, values[0] = 2, 2 % 2 = 0 and i % 2 = 0. Good number.
    • 

  • For i = 1, values[1] = 3, 3 % 2 = 1 and i % 2 = 1. Good number.
    • 

  • For i = 2, values[2] = 4, 4 % 2 = 0 and i % 2 = 0. Good number.
    • 

  • For i = 3, values[3] = 5, 5 % 2 = 1 and i % 2 = 1. Good number.
    • 



Sum of elements added: 2 + 3 + 4 + 5 = 14.
Hence, the answer is 14.




Constraints


    

  • 1 <=nums.size()<= 106
    • 

  • 1 <=nums[i]<= 1000
    • 

  • Index starts from 0
    • 



def sum_good_number(n, nums):
    a=0
    for i in range(n):
        if nums[i]%2 == i%2:
            a=a+nums[i]
    return(a)
