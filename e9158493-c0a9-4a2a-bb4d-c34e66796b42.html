Password Encoding


You are given a password string P of length N. The password needs to be encoded before storing it to the database. The password string can be encoded using the following rules:


    

  • If the k-th character in P is a vowel, change it to the second next character in the alphabetical sequence. For example, the second next character of o is q.
    • 

  • If the k-th character in P is a consonant, change it to the second previous character in the alphabetical sequence. For example, the second previous character of h is f.
    • 



Implement the function encode which will take password string P as argument and return encoded string.



1) The second previous character of b is z.



Note: You need not to write code to handle input and output.




Input Format


    

  • Single line containing password string.
    • 





Output Format


    

  • Single line containing encoded password string.
    • 





Example 1


Input


abc



Output


cza



Explanation


a is vowel so change to c
b is consonant so change to z
c is consonant so change to a
So, encoded password for abc will be cza

iloveniat



Output


kjqtglkcr



Explanation

l is consonant so change to j

n is consonant so change to l
i is vowel so change to k
a is vowel so change to c
t is consonant so change to r
So, encoded password for iloveniat will be kjqtglkcr




Constraints


    

  • 1 <=N<= 106
    • 

  • The password string P contains only lowercase alphabets.
    • 



def encode(P) :
    S="abcdefghijklmnopqrstuvwxyz"
    result=""
    for i in P:
        if i=="a" or i=="e" or i=="i" or i=="o" or i=="u":
            index=S.index(i)
            result+=S[index+2]
        else:
            index=S.index(i)
            result+=S[index-2]
    return(result)

Difference of Multiples


For a list of integers nums, nums[i] is said to be a Multiple if it is divisible by (i + 1)
Implement the function difference_of_multiples which will take n, size of list and nums as argument and return an integer, the absolute difference between the sum of multiples and the sum of non-multiples.


Note: You need not to write code to handle input and output.




Input Format


    

  • First line contains an integer n, the size of nums.
    • 

  • Second line contains n space-separated integers, numbers in list.
    • 





Output Format


    

  • One line containing an integer, the absolute difference between sums.
    • 





Example 1


Input


4
2 3 6 8



Output


13



Explanation


    

  • For i = 0, nums[0] = 2, divisible by 1. Multiple.
    • 

  • For i = 1, nums[1] = 3, not divisible by 2. Non-multiple.
    • 

  • For i = 2, nums[2] = 6, divisible by 3. Multiple.
    • 

  • For i = 3, nums[3] = 8, divisible by 4. Multiple.
    • 



Sum of multiples: 2 + 6 + 8 = 16.
Sum of non-multiples: 3.
Absolute difference: |16 - 3| = 13.




Constraints


    

  • 1 <= nums.size() <= 106
    • 

  • -1000 <= nums[i] <= 1000
    • 

  • Index starts from 0
    • 



def difference_of_multiples(n, nums):
    Multiples=0
    Non_Multiples=0
    for i in range(n):
        if nums[i]%(i+1)==0:
            Multiples+=int(nums[i])
        else:
            Non_Multiples+=int(nums[i])
    if Multiples>Non_Multiples:
        return(Multiples-Non_Multiples)
    else:    
         return(Non_Multiples-Multiples)

Element W is Close Pair if there exists at least 1 element whose absolute difference with W is less than or equal to H. An element W is Close Pair, if there is another element in the range [W-H, W+H] other than W itself.




Input Format


    

  • First line contains two integers Q and H respectively.
    • 

  • Second line contains Q integers separated by space.
    • 





Output Format


    

  • Print a single integer denoting the total number of close pair elements.
    • 





Constraints


    

  • 1 <=Q<= 1000
    • 

  • 0 <=H<= 1000
    • 

  • 0 <=E[i]<= 1000
    • 





Explanation


Let's take an example:


4 5
10 21 13 14



arr = [10, 21, 13, 14]


For W=10 to be Close Pair, 
there should be atleast one element in range: [10-5, 10+5] = [5, 15].
So, this is a Close Pair. 


For W=21 to be Close Pair, 
there should be atleast one element in range: [21-5, 21+5] = [16, 26].
There is no element in this range.
So, this is a not Close Pair.


Similarily,
13 and 14 are Close Pair 
Hence we have 3 Close Pair.

1. Each string consists of lowercase English alphabets only.

Input Format


    

  • The first line contains an integer n, the number of strings in the array.
    • 

  • The second line contains n space-separated strings, representing strArr.
    • 



Output Format


    

  • Print the number of pairs of strings that share at least one character.
