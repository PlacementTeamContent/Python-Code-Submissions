Develop a Functional Learner Class


Implement a class Learner with the following specifications.


Class Description


Learner


    

  • Attributes  
    • 

  • full_name – Full name of the learner.  
    • 

  • learner_id – Unique learner ID.  
    • 

  • course – Course the learner is enrolled in.  
    • 

  • score – Learner's score (default is 0.0).  
    • 

  • 

    completed – A boolean indicating whether the learner has completed the course (default should be False).  
    

    • 

  • 

    Methods
    

    • 

  • get_details() – Returns a string combining the full name, ID, and course in the format:
    
    "[full_name] (ID: [learner_id]) is enrolled in [course]"

    "John Doe (ID: 67890) is enrolled in Data Science"
    • 

  • mark_as_completed() – Sets [completed] to True and prints:
    
    "Learner [full_name] has completed the course"  
    • 

  • update_score(new_score) – Updates the score to new_score and prints:
    
    "Score updated to [new_score] for [full_name]"  
    • 



 
1. The init method should take 3 arguments: full_name, learner_id, and course.

2. The attribute score should start at 0.0, and completed should start as False.

3. The implementation should be tested for correctness.

Versatile Financial Account System


Design a financial account management system with a base class FinancialAccount and two subclasses: SavingsAccount and CreditAccount.


Class Design


FinancialAccount


    

  • Attributes
    • 

  • owner_name – Name of the account holder.
    • 

  • balance – Current account balance.
    • 

  • transactions – List to record all transaction details.
    • 

  • Methods
    • 

  • deposit(amount) – Increases balance and logs "Deposited: [amount]".
    • 

  • withdraw(amount) – Deducts amount if sufficient, logs "Withdrew: [amount]", else prints "Insufficient balance".
    • 

  • get_balance() – Returns the current balance.
    • 

  • view_transactions() – Displays the transaction history.
    • 



SavingsAccount (inherits from FinancialAccount)


    

  • Additional Attributes
    • 

  • interest_rate – Annual interest rate (e.g., 3.5 for 3.5%).
    • 

  • Additional Methods
    • 

  • apply_interest() – Uses:
    
    interest = balance * interest_rate / 100
    
    Adds the interest to balance and logs "Interest Applied: [interest]".
    • 



CreditAccount (inherits from FinancialAccount)


    

  • Additional Attributes
    • 

  • credit_limit – Maximum negative balance allowed (e.g., 7000).
    • 

  • Overridden Methods
    • 

  • withdraw(amount) – Allows withdrawal within credit_limit, otherwise prints "Credit limit exceeded".
    • 

  • If balance becomes negative, logs "Withdrew: [amount] (Credit Used)".
    • 




1. FinancialAccount constructor initializes owner_name, balance, and an empty transactions list.
2. SavingsAccount requires interest_rate in initialization.
3. CreditAccount requires credit_limit in initialization.
4. Every transaction must be logged.
5. Proper method overriding and implementation will be evaluated.

Bus (Base Class)


    

  • Private Attributes:
    • 

  • __bus_id – The unique identifier for the bus.
    • 

  • __bus_name – The name of the bus.
    • 

  • 

    __base_fare – The base fare for the bus per passenger.
    

    • 

  • 

    Public Methods:
    

    • 

  • display_info(self) – This method should print the bus ID, name, and base fare of the bus in the format: Bus ID: [bus_id], Bus Name: [bus_name], Base Fare: $[base_fare]
    • 

  • calculate_fare(self, passengers) – This method should calculate and return the total fare based on the number of passengers. The base fare is multiplied by the number of passengers, and adjustments are made based on the bus type.
    • 





LuxuryBus (inherits from Bus)


    

  • Additional Attributes:
    • 

  • 

    __amenities – A list of amenities available in the luxury bus.
    

    • 

  • 

    Overrides calculate_fare method:
    

    • 

  • The fare for a luxury bus is calculated as:
    Total Fare = Base Fare * Passengers * 1.8
    • 

  • Luxury buses charge 1.8x the base fare due to premium amenities.
    • 





CityBus (inherits from Bus)


    

  • Additional Attributes:
    • 

  • 

    __routes – The number of routes the city bus covers.
    

    • 

  • 

    Overrides calculate_fare method:
    

    • 

  • The fare for a city bus is calculated as:
    Total Fare = Base Fare * Passengers * 1.1
    • 

  • City buses charge 1.1x the base fare due to frequent stops and accessibility features.
    • 






1. The init method in Bus should initialize __bus_id, __bus_name, and __base_fare.
2. The calculate_fare(self, passengers) method in Bus should return the basic fare multiplied by the number of passengers if not overridden by subclasses.
3. Testing will verify that polymorphism is correctly implemented and that method overriding works as specified.

Digital Banking System with Abstraction


Design an abstract class DigitalBankAccount that acts as a blueprint for different types of digital bank accounts. Implement two subclasses: FixedDepositAccount and CheckingAccount.


DigitalBankAccount (Abstract Class)


    

  • Private Attributes:
    • 

  • __account_id – Unique identifier for the digital bank account.
    • 

  • __owner_name – Name of the account owner.
    • 

  • 

    __funds – Current available funds in the account.  
    

      

    • If __funds is negative, print "Invalid Funds".
      • 

    

    • 

  • 

    Abstract Methods:
    

    • 

  • add_funds(amount) – Must be implemented by subclasses.
    • 

  • remove_funds(amount) – Must be implemented by subclasses.
    • 





FixedDepositAccount (inherits from DigitalBankAccount)


    

  • Additional Private Attributes:
    • 

  • 

    __maturity_period – The period (in months) until funds can be withdrawn.  
    

      

    • If __maturity_period is zero or negative, print "Invalid Maturity Period".
      • 

    

    • 

  • 

    Implements Abstract Methods:
    

    • 

  • add_funds(amount) – Adds the amount to the funds and prints: "Fixed Deposit Account [account_id]: Deposited $[amount]. Total Funds: $[funds]."
    • 

  • remove_funds(amount) – Allows withdrawal only if the maturity period has passed. If eligible, prints: "Fixed Deposit Account [account_id]: Withdrawn $[amount]. Remaining Funds: $[funds]." Otherwise, prints: "Withdrawal denied. Maturity period not completed."
    • 





CheckingAccount (inherits from DigitalBankAccount)


    

  • Additional Private Attributes:
    • 

  • 

    __transaction_limit – The maximum amount allowed per transaction.  
    

      

    • If __transaction_limit is negative, print "Invalid Transaction Limit".
      • 

    

    • 

  • 

    Implements Abstract Methods:
    

    • 

  • add_funds(amount) – Adds the amount to the funds and prints: "Checking Account [account_id]: Deposited $[amount]. Available Funds: $[funds]."
    • 

  • remove_funds(amount) – If the amount is within the transaction limit and sufficient funds are available, prints: "Checking Account [account_id]: Withdrawn $[amount]. Remaining Funds: $[funds]." Otherwise, prints: "Transaction limit exceeded or insufficient funds."
    • 






1. The init method in DigitalBankAccount should initialize account_id, __owner_name, and __funds.
2. The __init methods in FixedDepositAccount and CheckingAccount should initialize inherited attributes and apply their own logic.
3. The DigitalBankAccount class must not be directly instantiated.
4. The implementation will be verified for abstraction, inheritance, and method overriding.
