Class Description


Account


    

  • Attributes
    • 

  • account_holder – Name of the account owner.
    • 

  • balance – Current account balance (float).
    • 

  • transaction_log – A list to maintain transaction history.
    • 

  • Methods
    • 

  • deposit(amount) – Increases balance by amount and logs "Deposit: [amount]".
    • 

  • withdraw(amount) – Deducts amount if sufficient balance, logs "Withdrawal: [amount]", else logs "Insufficient balance".
    • 

  • get_balance() – Returns the current balance.
    • 

  • view_transactions() – Displays all recorded transactions.
    • 



SavingsAccount (inherits from Account)


    

  • Additional Attributes
    • 

  • interest_rate – Fixed interest rate (e.g., 4.2 for 4.2% per annum).
    • 

  • Additional Methods
    • 

  • apply_interest() – Uses:
    
    interest = balance * interest_rate / 100
    
    Adds the computed interest and logs "Interest Added: [interest_amount]".
    • 



CheckingAccount (inherits from Account)


    

  • Additional Attributes
    • 

  • overdraft_protection – Maximum overdraft limit allowed (e.g., 1500).
    • 

  • Overridden Methods
    • 

  • withdraw(amount) – Allows overdraft up to overdraft_protection, else logs "Overdraft limit exceeded".
    • 

  • If the balance goes negative, logs "Overdraft Withdrawal: [amount]".
    • 




1. Account initializes account_holder, balance (default 0), and an empty transaction_log list.
2. SavingsAccount requires interest_rate during initialization.
3. CheckingAccount requires overdraft_protection during initialization.
4. Transaction logs should be properly maintained.
5. Proper use of method overriding will be assessed.
