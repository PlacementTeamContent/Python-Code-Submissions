Good Number Sum


For a list of integers nums, nums[i] is said to be Good Number if parity of i and nums[i] will be same.


Implement the function sum_good_number which will take n, size of list and nums as argument and return an integer, sum of good number in the list.


Note: You need not to write code to handle input and output.




Input Format


    

  • First line contains an integer n, the size of nums.
    • 

  • Second line contains n space-separated integers, numbers in list.
    • 





Output Format


    

  • One line containing an integer, the sum of good numbers.
    • 





Example 1


Input


3
1 2 3



Output


0



Explanation


    

  • For i = 0, nums[0] = 1, 1 % 2 = 1 and i % 2 = 0. Not a good number.
    • 

  • For i = 1, nums[1] = 2, 2 % 2 = 0 and i % 2 = 1. Not a good number.
    • 

  • For i = 2, nums[2] = 3, 3 % 2 = 1 and i % 2 = 0. Not a good number.
    • 



Hence, the answer is 0.


Example 2


Input


4
2 3 4 5



Output


14



Explanation


    

  • For i = 0, values[0] = 2, 2 % 2 = 0 and i % 2 = 0. Good number.
    • 

  • For i = 1, values[1] = 3, 3 % 2 = 1 and i % 2 = 1. Good number.
    • 

  • For i = 2, values[2] = 4, 4 % 2 = 0 and i % 2 = 0. Good number.
    • 

  • For i = 3, values[3] = 5, 5 % 2 = 1 and i % 2 = 1. Good number.
    • 



Sum of elements added: 2 + 3 + 4 + 5 = 14.
Hence, the answer is 14.




Constraints


    

  • 1 <=nums.size()<= 106
    • 

  • 1 <=nums[i]<= 1000
    • 

  • Index starts from 0
    • 



def sum_good_number(n, nums):
    d=0
    for i in range(n):
        if int(nums[i])%2==i%2:
            d+=int(nums[i])
    return d

Total Calories Consumed


Implement a function total_calories to determine the total calories consumed after T meals, starting with an initial calorie intake of C calories. Each subsequent meal adds a constant number of A calories.


Function total_calories will take C, A, and T as first, second, and third arguments respectively and return the total calories consumed after T meals.


Note: You need not write code to handle input and output.




Input Format


    

  • Single line containing 3 space-separated integers representing C, A, and T respectively.
    • 





Output Format


    

  • Single line having 1 integer representing the total calories consumed after T meals.
    • 





Example 1


Input


500 200 4



Output


1300



Explanation


Initial calories consumed is 500.
At meal 1, total calories will be 500+200 = 700.
At meal 2, total calories will be 700+200 = 900.
At meal 3, total calories will be 900+200 = 1100.
At meal 4, total calories will be 1100+200 = 1300.




Constraints


    

  • 0 <= C, A, T < 5000
    • 



def total_calories(C, A, T):
    M=int(C)
    for i in range(int(T)):
        M+=int(A)
    return M

def encrypt(M):
    d=""
    vowels="aeiou"
    for i in M:
        if i in vowels:
            d+=chr(ord(i)+1)
        else:
            d+=chr(ord(i)-1)
    return d

Input Format

Output Format

Constraints

Explanation


Let's take an example:


4 5
10 21 13 14



arr = [10, 21, 13, 14]


For V=10 to be proximate element, 
there should be atleast one element in range: [10-5, 10+5] = [5, 15].
So, this is a proximate element. 


For V=21 to be proximate element, 
there should be atleast one element in range: [21-5, 21+5] = [16, 26].
There is no element in this range.
So, this is a not proximate element.


Similarily,
13 and 14 are proximate element 
Hence we have 3 proximate element.
