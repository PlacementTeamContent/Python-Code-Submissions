Count Distinct Years in Document


The United Nations Organization released an official document regarding the most important events from the beginning of time (dated 00-00-0000) with a brief description of the events. The date of all the events is mentioned in the 'DD-MM-YYYY' format.

Your task is to write a program to find the total number of distinct years referenced in the document.




Input Format


    

  • The first line of input contains a string containing the content of the document.
    • 





Output Format


    

  • Print the total number of distinct years referenced in the document.
    • 





Constraints


    

  • The length of the input string will be between 1 and 1000 characters.
    • 

  • The document string will contain dates in the 'DD-MM-YYYY' format.
    • 

  • The string will not contain invalid date formats.
    • 

  • The year part of the date (YYYY) will always be a valid 4-digit year.
    • 





Example


Sample Input


UN was established on 24-10-1945. India got freedom on 15-08-1947.


Sample Output


2


Explanation


    

  • Two distinct years, 1945 and 1947, have been referenced in the document.
    • 



import java.util.*;
class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        String str;
        int c = 0;
        str = sc.nextLine();
        for (int i = 0; i < str.length(); i++){
            if((str.charAt(i) == '1' && str.charAt(i+1) == '9') ||(str.charAt(i) == '2' && str.charAt(i+1) == '0')){
                c += 1;
            }
        }
        
        System.out.println(c);
    }
}
2.
Number of Chocolates


Calculate Maximum Number of Chocolates One Can Eat and Number of Wrappers Left in Hand


Given the following inputs, determine:
1. The maximum number of chocolates one can eat.
2. The number of wrappers left in hand.


You can buy chocolates with the money you have and then exchange wrappers for additional chocolates based on the rules provided. If a maximum number of shop visits is defined, the process should stop after that number of visits. Otherwise, the process continues until wrappers are insufficient for exchange.


Inputs:


    

  • Money: Total money one has to spend.
    • 

  • Price: Price per chocolate.
    • 

  • Wrappers: Minimum number of wrappers required for an exchange.
    • 

  • Choco: Number of chocolates received for exchanging wrappers.
    • 

  • Max visit: Maximum number of times one can visit the shop. (If zero, consider it infinite).
    • 



Input Format:


    

  • The input will be managed within the function itself. There is no need to write code for reading inputs separately.
    • 





Output Format:


    

  • Return total number of chocolates that can be eaten and the numbers of Wrappers left in hand.
    • 





Sample Input:


40
1
3
1
1




Sample Output:


53
14




Explaination


    

  • Initial Purchase:
    • 



Money = 40, Price = 1
  Initial Chocolates = 40//1 = 40
  Initial Wrappers = 40


    

  • First Visit:
    • 



Wrappers Required = 3, Chocolates per Wrapper Exchange = 1
  New Chocolates = 40//3 = 13
  Total Chocolates = 40+13 = 53
  Remaining Wrappers = 40%3+13 = 14


    

  • Maximum Visits Reached:
    • 



Max visit = 1, process stops here.


    

  • Final Output:
    • 



Total chocolates eaten = 53
  Wrappers left in hand = 14


public class Solution {

    public static void calculateChocolates(int money, int price, int wrappers, int choco, int maxVisit, int[] result) {
        int initialChocolates = (int)money / price;
        int initialWrappers = initialChocolates;
        for (int i = 0; i < maxVisit; i++){
            int newChocolates = ((int)initialWrappers / wrappers) * choco;
            int totalChocolates = initialChocolates + newChocolates;
            int remChocos = initialChocolates%wrappers + newChocolates;
            
            initialChocolates = totalChocolates;
            initialWrappers = remChocos;
            result[i] = totalChocolates;
        }
        result[0] = initialChocolates;
        result[1] = initialWrappers;
            
    }
}
3.
k-Anagram


Two strings are called k-anagrams if both of the below conditions are true:
1. Both have the same number of characters.
2. Two strings can become anagrams by changing at most k characters in a string.


Given two strings of lowercase alphabets and an integer k, implement a function to determine if the two strings are k-anagrams of each other.




Input Format


    

  • A string s1 consisting of lowercase English letters.
    • 

  • A string s2 consisting of lowercase English letters.
    • 

  • An integer k.
    • 



Output Format


    

  • Print true if s1 and s2 are k-anagrams, otherwise print false.
    • 





Constraints


    

  • 1 ≤ s1.size(), s2.size() ≤ 10⁵
    • 

  • 1 ≤ k ≤ 10⁵
    • 





Example 1


Input


fodr
gork
2


Output


True


Explanation


We can change 'f' → 'g' and 'd' → 'k' in s1 to make it an anagram of s2.




Example 2


Input


geeks
eggkf
1


Output


False


Explanation


We can modify only 1 character, but 2 modifications are needed ('g' and 'f' in s2).


import java.util.*;

class Solution {
    public boolean areKAnagrams(String s1, String s2, int k) {
        int n = s1.length();
        if (s1.length() == s2.length()){
            for (int i = 0; i < n; i++){
                boolean isEqual = true;
                for (int j = 0; j < n; j++){
                    if (s1.charAt(i) == s2.charAt(j)){
                        isEqual = false;
                    }
                }
                if (isEqual){
                    k -= 1;
                }
            }
            if (k==0){
                return true;
            }else{
                return false;
            }
        }else{
            return false;
        }
         
    }
}
