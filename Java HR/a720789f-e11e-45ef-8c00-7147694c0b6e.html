Borrow Number


You have two numbers number1 and number2. Your task is to check the number of borrow operations needed for the subtraction of number1 from number2. If the subtraction is not possible, print the string "Not possible".




Input Format


    

  • Two integers number1 and number2.
    • 



Output Format


    

  • Print the number of borrow operations if the subtraction is possible.
    • 

  • Print the string "Not possible" if the subtraction is not possible.
    • 





Constraints


    

  • number1 and number2 are positive integers.
    • 





Sample Input 0:


754
658


Sample Output 0:


2


Explanation:


    

  • First iteration (rightmost digit): 4 - 8 requires borrow
    • 

  • Second iteration: 5 - 5 (after first borrow) requires another borrow
    • 

  • Total borrow operations: 2
    • 



Sample Input 1:


654
666


Sample Output 1:


Not possible


Explanation:


    

  • the subtraction is not possible because 666 is greater than 654.
    • 



import java.util.*;
class Main {
    public static void main(String[] args) {
        // System.out.println("Be the programmer, you're meant to be!");
        Scanner sc=new Scanner(System.in);
        int num1=sc.nextInt();
        int num2=sc.nextInt();
        if(num2>num1){
            System.out.println("Not possible");
        }
        else if(num1==num2){
            System.out.println(0);
        }
        else if(num2<num1){
            int borrow=0;
            int temp=0;
            while(num1>0&&num2>0){
                int rem1=num1%10-temp;
                int rem2=num2%10;
                if(rem1<rem2){
                    borrow++;
                    temp=1;
                }
                else{
                    temp=0;
                }
                num1=num1/10;
                num2=num2/10;
            }
            System.out.println(borrow);
        }
    }
}

Probability of Getting Exactly R Heads


Write a program that will take two integers as input: 


    

  • N: The number of successive tosses of a fair coin.  
    • 

  • R: The exact number of heads required.  
    • 



The program should return the probability of getting exactly R heads in N successive tosses of a fair coin. A fair coin has an equal probability of landing a head or a tail (i.e., 0.5) on each toss.





Input Format


    

  • Two space-separated integers: N (number of tosses) and R (required number of heads).
    • 






Output Format


    

  • A floating-point number representing the probability of getting exactly R heads, rounded to six decimal places.
    • 






Constraints


    

  • 1 <= N <= 20  
    • 

  • 0 <= R <= N  
    • 






Example 1


Input


1 1


Output


0.500000


Explanation


    

  • There is only one toss, and the probability of getting a head in one toss is 0.5.
    • 





Example 2


Input


4 3


Output


0.250000


Explanation


    

  • The number of ways to get exactly 3 heads in 4 tosses is 4.  
    • 

  • Total possible outcomes for 4 tosses = 2⁴ = 16.  
    • 

  • Probability = 4/16 = 0.25.
    • 



import java.util.Scanner;

public class main {
