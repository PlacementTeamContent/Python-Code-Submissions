LRU Cache Implementation


You are tasked with implementing an LRU (Least Recently Used) Cache. The cache should support the following operations:


    

  • get(key): Retrieve the value associated with the key if it exists in the cache; otherwise, return -1.
    • 

  • put(key, value): Insert or update the value associated with the key. If the cache reaches its capacity, it should evict the least recently used item.
    • 



The solution should maintain O(1) time complexity for both operations.




Input Format


    

  • capacity: An integer representing the maximum number of items the cache can store.
    • 

  • A sequence of operations, where each operation is one of the following:
    • 

  • get(key) to retrieve a value.
    • 

  • put(key, value) to insert or update a key-value pair in the cache.
    • 





Output Format


    

  • For each get(key) operation, return the value associated with the key or -1 if the key does not exist in the cache.
    • 

  • No output is required for put(key, value) operations.
    • 





Example


Input:


 capacity = 2
 operations = ["put(1, 1)", "put(2, 2)", "get(1)", "put(3, 3)", "get(2)", "put(4, 4)", "get(1)", "get(3)", "get(4)"]




Output:


 1
 -1
 -1
 3
 4




Explanation:


    

  • put(1, 1) and put(2, 2): Add two items to the cache.
    • 

  • get(1): Returns 1 because the key exists in the cache.
    • 

  • put(3, 3): Evicts the least recently used key 2 and adds key 3.
    • 

  • get(2): Returns -1 because key 2 was evicted.
    • 

  • put(4, 4): Evicts the least recently used key 1 and adds key 4.
    • 

  • get(1): Returns -1 because key 1 was evicted.
    • 

  • get(3) and get(4): Return 3 and 4, respectively, as they exist in the cache.
    • 

import java.util.*;

public class Solution {
    private final int capacity;
    private final Map<Integer, Integer> cache;

    public Solution(int capacity) {
        // Initialize the cache with the given capacity
        this.capacity = capacity;
        this.cache = new LinkedHashMap<>(capacity, 0.75f, true){
        protected boolean removeEldestEntry(Map.Entry<Integer,Integer> eldest){
            return size() > Solution.this.capacity;
        }
        };
    }

    public int get(int key) {
        // Implement the get operation
        return cache.getOrDefault(key,-1);
    }

    public void put(int key, int value) {
        // Implement the put operation
        cache.put(key,value);
    }
}

First Bench


In a class of n students, the teacher distributed answer sheets to the students. Few students have
scored very good marks but few have scored very fewer marks. The teacher is not happy with the
performance of the students in the test, so she wants to change the seating arrangement a bit. She
wants students scoring less to sit with students who score good marks so that they could get some help.
So, to begin with, she wants that on the first bench two students should sit, such that the difference
between the marks of those two students comes out to be maximum.


Students have got their marks and the marks of students have been arranged in the array where Mi 
determines the marks of the ith student.Your task is to determine the maximum difference
of marks that can be achieved between the students followed by how many such pairs are in the class
that follows the conditions mentioned above and are eligible to sit on the first bench




Input Format


    

  • First line of input contains the number of students 
    • 

  • Next line contains contains the marks of students
    • 



Output Format


    

  • The output should be maximum difference of marks
followed by the number of pairs possible with that
difference seperated by space.
    • 





Sample Input:


3
2 5 6


Sample Output:


4 1


Explanation:


The maximum mark difference is between 1st and 3rd  students, which is 6-2=4 And the valid pair thus can be only (2,6) thus, the count is 1


import java.util.Scanner;

public class main {

    public static void main(String[] args) {
      //Write your code here
      Scanner sc=new Scanner(System.in);
      int n=sc.nextInt();
      int[] marks=new int[n];
      for(int i=0;i<n;i++){
          marks[i]=sc.nextInt();
      }
      int min=marks[0],max=marks[0],mincount=1,maxcount=1;
      for(int i=1;i<n;i++){
          if(marks[i]<min){
              min=marks[i];
          }else if(marks[i]==min){
              mincount++;
          }
          if(marks[i]>max){
              max=marks[i];
              maxcount=1;
          }else if(marks[i]==max){
              maxcount++;
          }
      }
      int diff=max-min;
      int pairs=(diff==0)?(n*(n-1))/2:mincount*maxcount;
      System.out.println(diff+" "+pairs);
    }
}

Number of Chocolates


Calculate Maximum Number of Chocolates One Can Eat and Number of Wrappers Left in Hand


Given the following inputs, determine:
1. The maximum number of chocolates one can eat.
2. The number of wrappers left in hand.


You can buy chocolates with the money you have and then exchange wrappers for additional chocolates based on the rules provided. If a maximum number of shop visits is defined, the process should stop after that number of visits. Otherwise, the process continues until wrappers are insufficient for exchange.


Inputs:


    

  • Money: Total money one has to spend.
    • 

  • Price: Price per chocolate.
    • 

  • Wrappers: Minimum number of wrappers required for an exchange.
    • 

  • Choco: Number of chocolates received for exchanging wrappers.
    • 

  • Max visit: Maximum number of times one can visit the shop. (If zero, consider it infinite).
    • 



Input Format:


    

  • The input will be managed within the function itself. There is no need to write code for reading inputs separately.
    • 





Output Format:


    

  • Return total number of chocolates that can be eaten and the numbers of Wrappers left in hand.
    • 





Sample Input:


40
1
3
1
1




Sample Output:


53
14




Explaination


    

  • Initial Purchase:
    • 



Money = 40, Price = 1
  Initial Chocolates = 40//1 = 40
  Initial Wrappers = 40


    

  • First Visit:
    • 



Wrappers Required = 3, Chocolates per Wrapper Exchange = 1
  New Chocolates = 40//3 = 13
  Total Chocolates = 40+13 = 53
  Remaining Wrappers = 40%3+13 = 14


    

  • Maximum Visits Reached:
    • 



Max visit = 1, process stops here.


    

  • Final Output:
    • 



Total chocolates eaten = 53
  Wrappers left in hand = 14


import java.util.*;
public class Solution {

    public static void calculateChocolates(int money, int price, int wrappers, int choco, int maxVisit, int[] result) {
        // Make your logic here
        int cs=money/price;
        int ws=cs;
        int visits=0;
        while((maxVisit==0||visits<maxVisit) && ws>=wrappers){
            int exchange=(ws/wrappers)*choco;
            ws=ws%wrappers+exchange;
            cs+=exchange;
            visits++;
        }
        result[0]=cs;
        result[1]=ws;
        // return result;
        // return {cs,ws};
        
    }
}
