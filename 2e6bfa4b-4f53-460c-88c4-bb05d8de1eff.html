Build a Tiered Bank Account System


Develop a class-based banking system that consists of a base class BankAccount and two derived classes FixedDepositAccount and CurrentAccount.


Class Description


BankAccount


    

  • Attributes
    • 

  • account_holder – The name of the person who owns the account.
    • 

  • balance – The current balance as a float.
    • 

  • history – A list that records all transactions, with each entry describing an action performed.
    • 

  • Methods
    • 

  • deposit(amount) – Increases the balance by amount and logs the transaction as "Deposited: [amount]".
    • 

  • withdraw(amount) – Deducts amount from the balance if funds are sufficient, logging the transaction as "Withdrew: [amount]". If funds are insufficient, prints "Insufficient balance" without modifying the balance.
    • 

  • get_balance() – Returns the account’s current balance.
    • 

  • show_history() – Displays all transactions in chronological order.
    • 



FixedDepositAccount (subclass of BankAccount)


    

  • Additional Attributes
    • 

  • interest_rate – The interest rate per annum (percentage, e.g., 4.5 for 4.5%).
    • 

  • Additional Methods
    • 

  • add_interest() – Computes interest using the formula:
    
    interest = balance * (interest_rate / 100)
    
    Adds the computed interest to the balance and records the transaction as "Interest Added: [interest_amount]".
    • 



CurrentAccount (subclass of BankAccount)


    

  • Additional Attributes
    • 

  • overdraft_limit – The maximum negative balance allowed (positive float, e.g., 1500).
    • 

  • Overridden Methods
    • 

  • withdraw(amount) – Allows withdrawals beyond the balance, provided that the balance does not fall below -overdraft_limit.  
      

    • If withdrawing amount would exceed -overdraft_limit, print "Overdraft limit reached" and do not process the withdrawal.
      • 

    • If the withdrawal results in a negative balance, log the transaction as "Withdrew: [amount] (Overdraft)".
      • 

    

    • 




1. The init method in BankAccount should initialize account_holder, balance, and set history as an empty list.
2. The init method in FixedDepositAccount should take an additional interest_rate parameter and call the parent constructor.
3. The init method in CurrentAccount should take an additional overdraft_limit parameter and call the parent constructor.
4. Ensure that deposits, withdrawals, and interest additions are correctly recorded in the history list.
5. The testing will confirm the correct behavior of attributes, methods, inheritance, and method overriding.
